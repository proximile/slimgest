# Security Policy

## Reporting a Vulnerability

If you have discovered a vulnerability in slimgest, please report it
**privately** via GitHub Security Advisories. Open
<https://github.com/proximile/slimgest/security/advisories/new> and
submit the report there — the maintainers can then work on a fix without
the issue being publicly visible until it has been resolved.

Please **do not** open a regular GitHub Issue or pull request for
security problems. Public Issues are visible to everyone immediately.

If the vulnerability is inherited from upstream
[gitingest](https://github.com/coderamp-labs/gitingest), please consider
also reporting it upstream so that all downstream forks benefit from the
fix.

## Supported versions

slimgest tracks the latest release on the `main` branch. Only the
current release receives security fixes.
